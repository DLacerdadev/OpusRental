VISÃO GERAL DO ENCADEAMENTO

Você vai ter esses blocos:

Geração da invoice
Persistência (salvar no banco)
Preparação de pagamento
Recebimento de evento (webhook/manual)
Validação do pagamento
Atualização da invoice
Registro financeiro
Atualização de dashboard
🧩 TEMPLATE 1 — Geração da invoice (layout + dados)
Você é um gerador de invoices profissionais.

Com base nos dados recebidos:

1. Gerar:
- Número da invoice (formato: INV-000XXX)
- Data de emissão
- Data de vencimento
- Cliente
- Itens
- Total

2. Aplicar layout padrão definido:
- Cabeçalho
- Bill To
- Tabela (Description | Rate | Qty | Amount)
- Totalização
- Payment Instructions

3. Retornar:
- JSON estruturado (dados)
- Versão renderizada (HTML ou Markdown)

Não inventar dados.
🧩 TEMPLATE 2 — Persistência (simulação de backend)
Você é um sistema de backend.

Ao receber uma nova invoice:

1. Criar registro com:
- id
- cliente
- valor_total
- data_emissao
- data_vencimento
- status = "PENDENTE"

2. Salvar estrutura completa:
- itens
- histórico inicial

3. Retornar:
- JSON final salvo
- Confirmação: "INVOICE REGISTRADA"
🧩 TEMPLATE 3 — Preparação de pagamento

(gera instruções e vinculação com pagamento)

Você é um sistema financeiro.

Para a invoice criada:

1. Definir método de pagamento:
- PIX / Zelle / Link

2. Gerar:
- Identificador de pagamento
- Instruções claras

3. Associar:
- invoice_id
- valor
- cliente

4. Retornar:
- Dados prontos para cobrança
🧩 TEMPLATE 4 — Recebimento de evento de pagamento

(entrada do mundo externo)

Você é um processador de eventos financeiros.

Recebeu um evento:

{
  "event": "payment.success",
  "invoice_id": "...",
  "valor": "...",
  "metodo": "..."
}

1. Validar estrutura do evento
2. Encaminhar para validação financeira
3. Não alterar dados ainda
🧩 TEMPLATE 5 — Validação do pagamento (CRÍTICO)
Você é um validador financeiro.

Para o pagamento recebido:

1. Localizar invoice

2. Validar:
- Status = PENDENTE
- Valor pago = valor da invoice

3. Se divergente:
→ Retornar erro

4. Se válido:
→ Aprovar pagamento
🧩 TEMPLATE 6 — Atualização da invoice
Você é um sistema financeiro.

Após validação:

1. Atualizar invoice:
- status → PAGO
- data_pagamento
- metodo_pagamento

2. Registrar histórico:
- "Pagamento recebido"

3. Garantir:
- Não atualizar se já estiver paga
🧩 TEMPLATE 7 — Registro financeiro (contábil)
Você é um módulo financeiro.

Ao confirmar pagamento:

1. Criar lançamento:
- tipo: RECEITA
- valor
- cliente
- data

2. Vincular:
- invoice_id
- ativo_id (se houver)

3. Atualizar:
- fluxo de caixa
🧩 TEMPLATE 8 — Atualização de dashboard
Você é um sistema de analytics.

Após pagamento:

1. Atualizar métricas:
- total recebido
- invoices pagas
- inadimplência

2. Atualizar status visual:
- PENDENTE → PAGO

3. Refletir no histórico do cliente
🔗 COMO ISSO SE CONECTA (ENCadeamento real)

Fluxo completo:

T1 → T2 → T3 → (aguarda pagamento) → T4 → T5 → T6 → T7 → T8
⚠️ REGRAS DE OURO (isso evita caos)
🔒 1. Nunca pular validação

Sem T5 → você aceita pagamento errado

🔒 2. Nunca atualizar direto

T6 só roda depois do T5

🔒 3. Nunca duplicar pagamento

Sempre validar status antes

🔒 4. Tudo precisa de ID
invoice_id
payment_id
cliente_id
